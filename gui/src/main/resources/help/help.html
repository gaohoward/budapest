How to Use Artemis Budapest

  • Introduction

    • This tool is used to browse various data files of an Artemis broker in order to find useful informations, as in debugging for example.

    For information about Artemis please go to the Artemis Website (https://activemq.apache.org/components/artemis/) .

    This GUI tool can handle hugh dat files without consuming much of the memory. It loads and displays log entries in page mode, each time only one page is loaded. It groups data files into projects so it can be viewed, analyzed with a clear structure.


  • Getting Started

    • First you need to create a project:

    Then open the project

  • Analyzing the data

    • On the right pane above the log entries there is a toolbar that provides tools for analyzing the log.

    The text field is for you to input some expressions that are used together with the buttons to its right.


  • The <&ndash and -> buttons

    • They can locate a record entry based on what you input in the text field. When you press either button, it searches each line and try to find if it contains the string in the text field. Based on which button you pressed, it searches forward ("->") or backward ("<&ndash").


  • The "R" button

    • The R button will created a filtered data file from the project based on the expression you input in the text field. The expression can be

    a simple word, for example error, or
    an 'and' expression, for example error & warn, or
    an 'or' expression, for example error | warn, or
    a 'not' expression, for example !error, or
    a combination of above, for example error & warn & !jboss

    spaces are ignored. The precedence of operators are
    ! > & > |
    however you can use braces "()" to change it.

    By default all filtering are NOT case sensitive. If you need case-sensitive matching, use -m switch, like
    -m ERROR & warn | Exception

    To use special characters like & or | in expressions, you need to escape it with backslash, example: Exception\&error
    it will match record lines who contain "Exception&error" (case insensitive).


  • The "X" buttons

    • The X button, when pressed, will filter the target records with java regular expressions. Any value in the text field will be taken and interpreted as Java Regular Expression. The filtered records contains entries from the targe log that has a match to the regular expression.



  • The "<" and ">" buttons

    • The two buttons are used to turn pages.


  • The Goto button (G)

    • The last button is goto button. It allows you to go to a specific line by its line number.


    note: to go to the last line, input -1.

    Enjoy!
